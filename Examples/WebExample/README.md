# WebExample

The reference embedding pattern for TerminalUI in a Bun-served browser
app. A real TerminalUI `App` is built for WASI and mounted onto a canvas
via the [`GUI/WebTUIGUI`](../../GUI/WebTUIGUI) host — there is no
terminal-emulator dependency.

If you want the marketing landing or the docs corpus, see the
[top-level Website](../../Website). This package stays small and
focused on the embedding contract.

## What this is

Two cooperating parts:

- **`TerminalApp/`** — a Swift package with a reusable `WebExampleScenes`
  library target plus a thin `WebExampleApp` executable launcher. The
  library is the same `App` declaration that ships in the SwiftUI demo;
  the launcher just calls `TerminalWASIAppRunner.run(WebExampleApp.self)`.
- **`src/`** — a Bun host that:
  1. invokes the Swift WASI build to produce
     `TerminalApp/dist/scene-manifest.json` and
     `TerminalApp/dist/assets/app.wasm`,
  2. serves them with COOP/COEP headers (required for
     `SharedArrayBuffer`-backed stdin),
  3. mounts `WebTUIGUI` against the manifest and wasm.

The whole pattern is ~60 lines of load-bearing code in
[`src/frontend.ts`](src/frontend.ts):

- `ensureCrossOriginIsolation()` — installs a service-worker COOP/COEP
  shim when Pages can't set headers itself
- `createWebTUIApp({ mount, manifestUrl, sceneRuntimeFactory, ... })` —
  the WebTUIGUI entry point
- `createWasmSceneRuntimeFactory(wasmUrl, ...)` — wires the WASI
  bootstrap that runs `WebExampleApp` inside each scene runtime

Everything else in `src/frontend.ts` is page chrome (header, scene
picker, status line, resize handle). A host page that adopts this
pattern can render whatever surrounding chrome it likes — only the
`.terminal-shell` element + the `data-*` hooks consumed by
WebTUIGUI are load-bearing.

## What to copy when adopting this pattern

The minimum a host needs:

1. A WASI build of a TerminalUI `App` (see `TerminalApp/`).
2. A static manifest the WebTUIGUI can read
   (`scene-manifest.json` — generated by `TerminalWASIAppRunner` in
   manifest mode).
3. The COI service-worker shim if hosting on a platform that can't set
   COOP/COEP headers directly (e.g. GitHub Pages).
4. `bun add webtuigui` in the host workspace.
5. The mount sequence from `src/frontend.ts:bootstrap()` —
   `createWebTUIApp` + `createWasmSceneRuntimeFactory`.

## Toolchains

- **Swift** (via `swiftly`): Swift 6.3.1 + the
  `swift-6.3.1-RELEASE_wasm` SDK. The WASI release build needs
  `-Xswiftc -Osize` plus
  `-Xswiftc -Xfrontend -Xswiftc -disable-llvm-merge-functions-pass`;
  plain `-O` (and on some Darwin runners even plain `-Osize`) emits
  merged outlined copy helpers whose signatures exceed the browser
  WebAssembly API's 1000-parameter limit and cause
  `WebAssembly.Module doesn't parse` startup failures. Stack and heap
  sizes also need configuring for non-trivial TerminalUI programs.
  The build script captures the canonical command.
- **Bun** for the web app, the bundler, and the test runner.

See the repo-level [`docs/TOOLCHAINS.md`](../../docs/TOOLCHAINS.md) for
the canonical environment story.

## Setup

```bash
bun install
```

`WebExample`, `GUI/WebTUIGUI`, and the top-level `Website` share the
repo's Bun workspace. Running `bun install` from the repo root is
preferred, but it also works from this directory.

## Development

```bash
bun dev
```

This first builds `TerminalApp/dist/scene-manifest.json` and
`TerminalApp/dist/assets/app.wasm`, then starts the Bun server.

The dev server runs Bun's HTML-import bundler behind a small proxy that
adds the COOP/COEP headers. Hot module reloading is disabled —
refresh the page after frontend edits.

## Production build

```bash
bun run build
bun run start
```

`bun run build` produces `dist/` (web bundle) and `pages-dist/`
(combined web bundle + `TerminalApp/dist/`). The repo's
[`pages-build.yml`](../../.github/workflows/pages-build.yml) workflow
ships `pages-dist/` to GitHub Pages under `/webexample/` so the
top-level Website can deep-link to it.

## File layout

```
Examples/WebExample/
├── TerminalApp/
│   ├── Package.swift
│   └── Sources/
│       ├── WebExampleScenes/      ← reusable App declaration
│       └── TerminalApp/main.swift ← thin WASI launcher
└── src/
    ├── index.html                 ← minimal page shell
    ├── index.css                  ← embedding stylesheet (.terminal-* + .example-*)
    ├── frontend.ts                ← bootstrap, mount, scene picker
    ├── app-data.ts                ← manifest paths + fallback manifest
    ├── scene-runtime.ts           ← WebTUIGUI scene runtime factory
    ├── wasm-scene-worker.ts       ← per-scene wasm worker
    ├── wasi-input-queue.ts        ← stdin shim for SharedArrayBuffer-backed input
    ├── browser-integration.browser.ts  ← browser-side glue
    ├── built-app-server.ts        ← production server
    ├── build-pages.ts             ← composes pages-dist/
    ├── build-terminal.ts          ← invokes the Swift WASI build
    ├── coi-serviceworker.js       ← COOP/COEP shim
    ├── docs-overrides.css         ← DocC theme overrides (consumed by pages-build.yml)
    ├── inject-docs-css.ts         ← DocC post-processor (consumed by pages-build.yml)
    └── start.ts                   ← `bun run start` entry
```
